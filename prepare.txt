Mapping & routing in compiler, initial mapping choose logic, IBM vs UR method"In a quantum compiler, mapping is the process of assigning logical qubits (from the circuit) to physical qubits on the hardware coupling map. Routing then inserts SWAP gates (or equivalent) whenever two logical qubits that need to interact are not directly connected.Initial mapping logic is critical because a bad starting assignment explodes the number of SWAPs later. The heuristic I use evaluates the interaction graph of the circuit (edges = two-qubit gates) against the hardware coupling graph, weighting by calibration data: higher error rates or longer decoherence on certain qubits get lower priority. We also consider future gate density — dense sub-circuits should map to high-connectivity regions.IBM's approach (in Qiskit) relies on SabreLayout + SabreSwap. SabreLayout does multiple random initial layouts + reverse-traversal trials to minimize estimated future SWAPs; then SabreSwap uses a heuristic lookahead for routing. It's fast and scalable but can get stuck in local minima on heavy circuits.In our UR method (the one I helped develop/refine), we go beyond random trials: we seed the initial mapping using a fidelity-aware VF2 subgraph isomorphism combined with a lookahead cost function that incorporates real-time T1/T2 and gate error data. This consistently reduces SWAP count by 15-30% over pure Sabre on heavy circuits I've benchmarked, especially on IBM Eagle/Heron devices, because we penalize noisy edges earlier instead of discovering them during routing."2. Task manager QASM to run actual hardware"To execute on real hardware, we first transpile the circuit to OpenQASM 2.0/3.0 (the hardware-native assembly). Then we submit it via the IBM Quantum task manager (or Qiskit Runtime service).The flow is:Authenticate → select backend (with its coupling map + calibration data).
Pass the QASM string + shots + optimization_level to the task manager.
It queues the job, runs calibration-aware transpilation if not pre-done, executes on the superconducting chip, and returns counts + metadata.

The task manager handles dynamic calibration fetching (latest gate errors, T1/T2) and error mitigation options (like ZNE or PEC). This abstraction lets us focus on circuit design while the manager deals with hardware scheduling and real-time noise."3. T1, T2, avg decoherence time"T1 (relaxation time) is the characteristic time for a qubit in |1⟩ to spontaneously decay to |0⟩ due to energy loss to the environment. T2 (dephasing time) is the time over which the relative phase between |0⟩ and |1⟩ is lost due to low-frequency noise (flux noise, charge noise, etc.). The effective coherence time that actually limits circuit depth is usually min(T1, T2), where T2 includes both dephasing and relaxation. Average decoherence time across the chip directly dictates maximum reliable circuit depth — on current IBM devices it's typically 50–150 µs for T1 and 20–100 µs for T2. In the compiler we use these values in the routing cost function so we never schedule long idle times on qubits with poor T1/T2."4. How superconducting qubits are made + QEC"Superconducting qubits (transmons) are fabricated on a high-resistivity silicon wafer using standard semiconductor processes:Deposit aluminum or tantalum for capacitor pads and Josephson junction.
Use double-angle evaporation + oxidation to create the Al-AlOx-Al Josephson junction (the non-linear inductor).
Wire it with coplanar waveguides or 3D cavities for control/readout.

The whole chip is cooled to ~10–20 mK in a dilution fridge to reach the superconducting regime.Quantum Error Correction (QEC) on these devices usually targets the surface code or heavy-hex code. We encode one logical qubit into many physical qubits (distance-d code needs d² physical qubits). Stabilizer measurements detect bit-flip and phase-flip errors in real time. Current devices are in the “QEC below threshold” regime — error rates per gate must stay below ~1% for the logical error to decrease exponentially with code distance. That's why hardware-native gates and low-latency classical control are so critical."5. Eigen values, characteristic eq, Hamiltonian matrix eigen values"The Hamiltonian matrix H describes the total energy of the quantum system (kinetic + potential terms). Its eigenvalues are the allowed energy levels (E₀, E₁, …) of the qubit or multi-qubit system.To find them we solve the characteristic equation det(H – λI) = 0. The eigenvectors give the corresponding stationary states. In superconducting qubits, the Hamiltonian is engineered so the two lowest eigenvalues correspond to the computational |0⟩ and |1⟩ basis — everything else is leakage. When we apply a drive (microwave pulse), we effectively rotate in this eigenbasis. Understanding this is fundamental because any gate we apply is a time evolution under a controlled Hamiltonian."6. RZ is why used mostly, noise is less why? loss decomposition why?"RZ (virtual Z-rotation) is the most used single-qubit gate because it is virtual — implemented purely by software phase tracking with zero microwave pulse. On superconducting hardware, we just update the phase reference frame of subsequent pulses; no actual control pulse → zero added noise, zero leakage risk, and gate error is essentially zero (machine precision).When we need arbitrary single-qubit rotations we decompose U(θ,φ,λ) into RZ + RX/RY, but we push as many RZ as possible to the end (or cancel them) because they cost nothing. 'Loss decomposition' refers to the fact that every time we decompose a gate into native gates we can introduce small unitary errors or extra depth — but with RZ we minimize that loss dramatically. This is why IBM and most platforms make RZ native and free."7. Bloch sphere how to rotate 90°, how it looks"The Bloch sphere is the geometric representation of a single qubit state: |ψ⟩ = cos(θ/2)|0⟩ + e^{iφ}sin(θ/2)|1⟩. North pole = |0⟩, south = |1⟩, equator = superpositions.To rotate 90° around X-axis (RX(π/2)): it takes |0⟩ → |+⟩ (equator) and |+⟩ → |1⟩. Visually, the state vector flips 90° from north pole straight down to the positive X-axis on the equator. Around Z (RZ): it just spins the vector around the vertical axis without changing latitude — pure phase change. Around Y does the orthogonal flip. I always visualize it as a unit vector on the sphere; any gate is a rotation operator on that sphere."8. Pulse shaping"In superconducting qubits we control via microwave pulses through control lines. Raw rectangular pulses have high-frequency components that cause leakage out of the computational subspace and excite higher transmon levels.Pulse shaping (Gaussian, DRAG, or cosine) suppresses these side lobes in the frequency domain. DRAG (Derivative Removal by Adiabatic Gate) specifically cancels the quadrature component that drives |1⟩  |2⟩ transitions. Good pulse shaping directly improves gate fidelity from ~99% to >99.9% and reduces decoherence during the gate. The compiler ultimately chooses pulse parameters based on the latest calibration."9. Native gates"Native gates are the set the hardware can execute directly with highest fidelity:On IBM Heron/Eagle: usually X, SX (√X), RZ(θ), ECR (echoed cross-resonance for two-qubit).
ECR is the native entangling gate; CX is synthesized from two ECR + single-qubit gates.
We never use non-native gates in the final circuit — the transpiler decomposes everything into these to minimize error and depth."

10. Pauli operators"Pauli operators {I, X, Y, Z} form the complete basis for any single-qubit operator and are the generators of rotations on the Bloch sphere.X = bit flip
Z = phase flip
Y = bit + phase
Any Hermitian observable can be written as a linear combination of Paulis. In stabilizer codes and QEC we measure Pauli strings; in simulation we use Pauli twirling for error mitigation. They are the alphabet of quantum error channels."

11. Pre, in, post trees, BFS, DFS"In the compiler front-end we build an Abstract Syntax Tree (AST) of the quantum circuit or QASM. Pre-order traversal: visit root → left → right → used for code generation (emit gate before recursing on operands).
In-order: left → root → right → useful for expression evaluation or printing.
Post-order: left → right → root → perfect for optimization passes that need to process children first (e.g., gate cancellation).

For the coupling graph during routing we run BFS (breadth-first) for shortest-path SWAP routes because it guarantees minimal depth. DFS can find paths but may give longer ones; we use it only when memory is a concern on huge chips."12. Hermitian matrix"A matrix H is Hermitian if H† = H (conjugate transpose equals itself). In quantum mechanics every observable is represented by a Hermitian operator because its eigenvalues are guaranteed real (measurable energies/values) and eigenvectors are orthogonal. The qubit Hamiltonian, Pauli matrices, and density-matrix evolution are all Hermitian — this is what guarantees unitary time evolution and real probabilities.""Dataset is not compiler" — if this came up, I'd say: "Exactly — a dataset of circuits is just benchmarking data.
